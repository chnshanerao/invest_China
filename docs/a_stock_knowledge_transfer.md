## 三、可迁移的分析框架

### 3.1 红绿灯信号系统（需适配A股指标）

**美股版本**:
| 维度 | 指标 | 绿灯 | 黄灯 | 红灯 |
|------|------|------|------|------|
| 恐慌度 | VIXY(VIX代理) | <20 | 20-35 | >35 |
| 利率 | TLT(美债反向) | >90 | 82-90 | <82 |
| 板块 | SOX(费半) | -3%~+3% | -5%~-2% | <-5% |

**A股适配建议**:
| 维度 | A股指标 | 说明 |
|------|---------|------|
| 恐慌度 | 50ETF隐含波动率(iVIX) / 融券余额变化 | A股无直接VIX产品 |
| 资金面 | 北向资金净流入(沪/深股通) | 外资风向标，>50亿=乐观 |
| 流动性 | SHIBOR / DR007 / 央行逆回购 | 替代TLT的利率信号 |
| 杠杆度 | 融资融券余额 | 融资余额快速上升=过热 |
| 板块 | 中证芯片指数(930891) / 半导体ETF(512480) | 替代SOX |
| 情绪 | 涨停家数 vs 跌停家数 / 换手率 | A股特有的情绪温度计 |

### 3.2 事件驱动日历框架

**美股已验证的模式**:
- 提前标记关键日期 + 预设动作
- 事件前减仓对冲 vs 事件后追踪确认
- 分批执行（不在单一事件all-in）

**A股需要跟踪的事件类型**:
| 事件类型 | 典型事件 | 频率 |
|---------|---------|------|
| 货币政策 | LPR报价(每月20日)、MLF操作、央行公开市场 | 月/周 |
| 经济数据 | 财新PMI(每月1日)、CPI/PPI、社融 | 月 |
| 财报季 | A股财报(4/8/10月底截止) | 季 |
| 政策面 | 国常会、政治局会议(4/7/10/12月) | 不定期 |
| 外部冲击 | 美联储FOMC、中美关系、关税 | 不定期 |
| 资金面 | MSCI/FTSE调仓、IPO节奏、限售解禁 | 季/月 |

### 3.3 分批建仓/减仓策略

**已验证的3批模式**:
- 第1批: 基础信号触发 → 试探性建仓(30%)
- 第2批: 宏观确认 → 加仓(30%)
- 第3批: 基本面验证 → 满配(40%)

**A股适配注意**:
- **T+1规则**: 当日买入次日才能卖出，所以止损必须提前设定，不能像美股日内操作
- **涨跌停板**: 主板±10%，科创板/创业板±20%，ST股±5%。涨停打不开=无法追高，跌停卖不出=止损失效
- **集合竞价**: 9:15-9:25决定开盘价，9:25-9:30不能撤单。这是A股独有的博弈窗口

### 3.4 止损纪律

**从07709事件学到的**:
- 杠杆产品必须有更严格的止损线（1x产品-20%，杠杆产品-15%甚至-10%）
- 系统自动检测 + 推钉钉告警 → 执行速度从"可能忘了"变成"分钟级"
- 去感情化: 不要因为"已经亏了这么多"就不忍心割，沉没成本不是理由

**A股止损特殊考虑**:
- 跌停板可能导致无法卖出 → 止损线要设在跌停前触发
- A股无盘后交易 → 所有操作必须在交易时段完成
- 可以用ETF替代个股来规避个股跌停风险

### 3.5 卡位理论(Chokepoint Theory)

**核心思想**: 找供应链中具有垄断/寡头地位的卡脖子环节

**美股实践**: ASML(唯一EUV光刻机) → TSM(CoWoS封装垄断) → MU(HBM存储)

**A股可应用的方向**:
- 半导体设备: 北方华创(刻蚀/薄膜)、中微公司(刻蚀)
- 半导体材料: 沪硅产业(大硅片)、南大光电(光刻胶)
- 封测: 长电科技、通富微电(先进封装)
- EDA: 华大九天、概伦电子
- AI算力: 寒武纪、海光信息
- 关键区别: A股的"卡位"不仅是市场地位，还包括"国产替代"政策红利

## 四、A股 vs 美股关键差异

| 维度 | 美股 | A股 |
|------|------|------|
| 交易制度 | T+0，可日内 | T+1，当日买入次日卖 |
| 涨跌限制 | 无限制(有熔断) | ±10%/±20%/±5% |
| 交易时间 | 09:30-16:00 + 盘前盘后 | 09:30-11:30, 13:00-15:00 |
| 做空机制 | 成熟(期权/融券/反向ETF) | 有限(融券门槛高) |
| 退市风险 | 低(机构主导) | 中(ST/退市规则频繁) |
| 信息效率 | 高(分析师覆盖广) | 中(内幕交易、游资炒作) |
| 主要驱动 | 基本面 + 流动性 | 政策面 + 资金面 + 情绪面 |
| 散户占比 | ~20% | ~60-70% |
| 分红税 | 按收入税率 | 持股>1年免税 |

## 五、A股监控系统技术架构（建议）

### 5.1 数据获取函数模板
```python
def parse_sina_a_stock(raw_line):
    """解析新浪A股数据行"""
    match = re.search(r'var hq_str_(s[hz]\d+)="(.+)"', raw_line)
    if not match or not match.group(2):
        return None
    symbol = match.group(1)
    parts = match.group(2).split(",")
    if len(parts) < 32:
        return None

    name = parts[0]
    open_price = float(parts[1]) if parts[1] != "0.000" else None
    prev_close = float(parts[2])
    price = float(parts[3])
    high = float(parts[4])
    low = float(parts[5])
    volume = int(float(parts[8]))  # 股
    turnover = float(parts[9])  # 元
    date = parts[30]
    time = parts[31]

    change_pct = (price - prev_close) / prev_close * 100 if prev_close > 0 else 0

    return {
        "symbol": symbol,
        "name": name,
        "price": price,
        "change_pct": change_pct,
        "prev_close": prev_close,
        "open": open_price,
        "high": high,
        "low": low,
        "volume": volume,
        "turnover": turnover,
        "timestamp": f"{date} {time}",
        "ok": True,
    }
```

### 5.2 A股交易时间调度
```
# 盘前: 09:00 — 获取集合竞价前的消息面/资金面
# 盘中: 09:45, 10:30, 11:15 — 上午扫描(3次)
# 午休: 12:30 — 检查北向资金半日数据
# 盘中: 13:15, 14:00, 14:45 — 下午扫描(3次)
# 收盘: 15:15 — 全量报告 + 日志
```

### 5.3 A股特有监控项
- **北向资金**: 实时监控沪/深股通净流入，大幅流出(>80亿)触发告警
- **融资余额**: 日度变化，连续增加=情绪升温
- **涨跌停统计**: 涨停<30只+跌停>50只 → 红灯
- **板块轮动**: 跟踪半导体/AI/新能源等板块ETF相对强弱
- **龙虎榜**: 游资/机构大额买卖

## 七、已知的技术坑（避坑指南）

1. **新浪API编码**: 返回GBK，必须用 `.decode("gbk", errors="replace")`
2. **SSL证书**: 某些环境需要跳过验证 `ssl.CERT_NONE`
3. **pip安装**: Devix容器PEP 668限制，需加 `--user --break-system-packages`
4. **Yahoo Finance**: 从此服务器返回403，不要用
5. **CloudCLI的cron格式**: 标准5字段cron格式
6. **钉钉Markdown**: 不支持表格，用列表+加粗替代
7. **新浪API频率**: 不要过于频繁调用（>1次/分钟），否则可能被封IP
8. **NoneType陷阱**: JSON state文件中的值可能是None，用 `(x or "")` 防御
